# TCS NQT Verbal Simulator

A production-grade AI-powered TCS NQT Verbal Ability assessment platform built with Next.js 15, Supabase, and Groq (Llama 3.3 70B).

## What it does

Simulates the real TCS NQT Verbal Ability exam with three sections:

1. **Fill in the Blanks** — 20 questions × 25 seconds each
2. **Passage Recall** — 4 passages × (30s read + 90s write)
3. **Email Writing** — 1 prompt × 540s · minimum 100 words

Every attempt is freshly generated by Groq Llama 3.3 70B. Passage recall and email writing are evaluated by AI on **meaning**, **key points**, **structure**, **grammar**, and **professionalism** — not memorization.

## Tech Stack

- **Framework**: Next.js 15 (App Router) + TypeScript
- **UI**: React 19, Tailwind CSS, shadcn-style components, Framer Motion, Lucide
- **State**: Zustand (with `persist` for session recovery)
- **DB + Auth**: Supabase (Postgres + Row Level Security)
- **AI**: Groq SDK + `llama-3.3-70b-versatile`

## Quick start

### 1. Install

```bash
yarn install
```

### 2. Configure environment

Copy `.env.example` to `.env.local` and fill in:

```env
NEXT_PUBLIC_SUPABASE_URL=...
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
SUPABASE_SERVICE_ROLE_KEY=...
GROQ_API_KEY=...
```

### 3. Set up the database

Open Supabase → SQL Editor and run the contents of [`supabase/schema.sql`](./supabase/schema.sql). This creates all tables, the auto-profile trigger, and Row Level Security policies.

### 4. Enable email/password auth

In your Supabase Dashboard → **Authentication → Providers → Email**, ensure **Email** is enabled.
For local testing, you can disable "Confirm email" so signups log in instantly.

### 5. Run

```bash
yarn dev
```

Open <http://localhost:3000>.

## Folder structure

```
app/
├── api/                       # Route handlers (Groq + Supabase)
│   ├── questions/{fill-blank,passage,email}/route.ts
│   ├── evaluate/{passage,email}/route.ts
│   └── attempts/[id]/complete/route.ts
├── dashboard/                 # Authenticated dashboard
├── exam/{section1,section2,section3}/  # Exam engine
├── results/[attemptId]/       # Post-exam results & feedback
├── login/  signup/            # Email/password auth pages
└── page.tsx                   # Landing page
components/ui/                 # shadcn-style UI primitives
lib/
├── supabase/                  # client + server + middleware helpers
├── groq/                      # Groq client + generators
└── evaluators/                # AI evaluation logic
store/exam-store.ts            # Zustand exam session store
supabase/schema.sql            # Database schema + RLS
middleware.ts                  # Protected routes
```

## Anti-cheat features

- Tab switch / visibility-change tracking → warning + DB log
- Right-click & copy/paste blocked during exam
- Text selection disabled (`exam-no-select`)
- Persistent session via Zustand (resume on refresh)
- Auto-submit on timeout for every section

## Deployment

Deploy on Vercel:

1. Push this project to GitHub.
2. Import the repo in Vercel.
3. Add the four env vars above in **Project Settings → Environment Variables**.
4. Deploy.
